Designing elegant onboarding flows in SwiftUI

A breakdown of the storyboard-to-SwiftUI migration strategy I used to build a smooth onboarding experience.

The Challenge

When I first started migrating my UIKit apps to SwiftUI, onboarding flows were one of the trickiest parts. The combination of page-style navigation, custom animations, and state management required a thoughtful approach.

Starting with UIPageViewController

In UIKit, I was used to relying on UIPageViewController for onboarding. It's powerful, but comes with a lot of boilerplate: 

class OnboardingViewController: UIPageViewController {
    var pages = [UIViewController]()
    
    override func viewDidLoad() {
        super.viewDidLoad()
        dataSource = self
        // ... more setup code
    }
}

Key Insights

1. State Management is Crucial

Using @AppStorage for tracking whether a user completed onboarding means you can check this state anywhere in your app without passing data around.

2. Custom Animations

One thing I love about SwiftUI is how easy it is to add custom transitions. Here's a fade-and-scale effect I used: 

.transition(.asymmetric(
    insertion: .scale(scale: 0.8).combined(with: .opacity),
    removal: .scale(scale: 1.2).combined(with: .opacity)
))

3. Skip and Continue Buttons

I implemented conditional button logic based on which page the user is viewing: 

HStack {
    if currentPage > 0 {
        Button("Back") {
            withAnimation { currentPage -= 1 }
        }
    }
    
    Spacer()
    
    Button(currentPage == pages.count - 1 ? "Get Started" : "Next") {
        if currentPage == pages.count - 1 {
            hasCompletedOnboarding = true
        } else {
            withAnimation { currentPage += 1 }
        }
    }
    .buttonStyle(.borderedProminent)
}

Performance Considerations

One issue I ran into was that all pages were being rendered immediately, even those not visible. For simple onboarding flows this isn't a problem, but if you have video or heavy images, consider lazy loading.

Final Thoughts

Migrating from UIKit to SwiftUI for onboarding taught me that sometimes the SwiftUI way requires unlearning old patterns. The declarative approach feels awkward at first, but once it clicks, you'll wonder how you lived without it.

If you're building onboarding flows, start simple with TabView, then layer in animations and custom logic. SwiftUI's composability makes it easy to refactor as you go.
